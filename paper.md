# Summary

Respiratory viruses circulate year-round and some have strong seasonal patterns. Consistently and reliable forecasts are essential for public health responder to provide clinical capacity planning, interventions and communication. ViroReportR is an R package that automates short-term forecasts and end-to-end routine report generation for common respiratory viruses (e.g. Sars-CoV-2, Influenza A and Respiratory Syncytial Virus) from routine surveillance data. Its implementation lies on using renewal equation to estimate the time-varying reproduction number \(R_t\) via EpiEstim package  [@coriNewFrameworkSoftware2013], and provides projections across time horizons (defined by the user) with validation metrics in a standardized HTML form. The tool is formulated for operational use by surveillance teams and public health officials, whose needs comprise robust, transparent, and reproducible results with minimal configuration. The package is available on CRAN and github with clear documentation, examples and vignettes showing the entire set up.

# Statement of need
Respiratory viruses causes recurrent stress in health system, especially during respiratory season. Short-term forecasts of cases are key epidemiological indicators to support public health decision-makers. Even though routine and reproducible multi-pathogen reports containing standardized uncertainty, and model validation are essential to build public trust, they are rarely provided alongside short-term forecasts. ViroReportR addresses this gap by providing an automated reporting workflow, where users supply daily counts in a three-column dataset (`date`, `disease_type`, `confirm`), optionally specify serial interval parameters, forecast horizon and seasonal patterns, and obtain a HTML report with forecasts plots, uncertainty intervals, \(R_t\) trajectories, and validation metrics across pathogens. Therefore, ViroReportR focuses on operational transparency and replicability, making it a well-suited and reliable tool to public health surveillance teams and policy workers who needs weekly briefs generated by routine data.

# Software design

We want to provide a flexible and easy to use report generation tool, maintaining the overall structure of the report fixed. ViroReportR package works around a main function `generate_forecast_report()` that calls all other functions for data processing, fitting, forecasting, validation and render a final markdown file (`inst/vriforecasting_report.Rmd`).
The data input is fairly simple and consists of the daily counts of a respiratory viruses. Given that jurisdictions might be interested in different respiratory viruses, we allow users to add or remove viruses. All of this can be done by the data input. The required format of the data is 3 columns with `date`, `disease_type` and `confirm` (confirmed cases). If the user has added any additional viruses other than Influenza A, SARS-CoV2 and RSV, they will also need to add serial interval distribution parameters for those viruses. The overall text and explanations on the report are not dependent on results. Adding new virus provides more tabs, plots and summary tables on the final html report.  In addition, some respiratory viruses follow a clear respiratory season, whereas others may not. Thus, we include an option for users to provide seasonal period for each virus if needed. Currently, we provide estimations using the `EpiEstim` packages [@coriNewFrameworkSoftware2013], but we design `ViroReportR` to allow for inclusion of other model techniques in the future.
