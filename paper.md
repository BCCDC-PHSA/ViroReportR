# Summary

Respiratory viruses circulate year-round and some have strong seasonal patterns. Consistently and reliable forecasts are essential for public health responder to provide clinical capacity planning, interventions and communication. ViroReportR is an R package that automates short-term forecasts and end-to-end routine report generation for common respiratory viruses (e.g. Sars-CoV-2, Influenza A and Respiratory Syncytial Virus) from routine surveillance data. Its implementation lies on using renewal equation to estimate the time-varying reproduction number \(R_t\) via EpiEstim package  [@coriNewFrameworkSoftware2013], and provides projections across time horizons (defined by the user) with validation metrics in a standardized HTML form. The tool is formulated for operational use by surveillance teams and public health officials, whose needs comprise robust, transparent, and reproducible results with minimal configuration. The package is available on CRAN and github with clear documentation, examples and vignettes showing the entire set up.

# Statement of need
Respiratory viruses causes recurrent stress in health system, especially during respiratory season. Short-term forecasts of cases are key epidemiological indicators to support public health decision-makers. Even though routine and reproducible multi-pathogen reports containing standardized uncertainty, and model validation are essential to build public trust, they are rarely provided alongside short-term forecasts. ViroReportR addresses this gap by providing an automated reporting workflow, where users supply daily counts in a three-column dataset (`date`, `disease_type`, `confirm`), optionally specify serial interval parameters, forecast horizon and seasonal patterns, and obtain a HTML report with forecasts plots, uncertainty intervals, \(R_t\) trajectories, and validation metrics across pathogens. Therefore, ViroReportR focuses on operational transparency and replicability, making it a well-suited and reliable tool to public health surveillance teams and policy workers who needs weekly briefs generated by routine data.

# State of the field                                              
There are a variety of packages that provides tools for surveillance, epidemic parameters estimation, nowcasting, or time-series forecasting. However, they usually target a different user groups and different layers of the analytic stack, and none provide a fully automated multi-pathogen reporting tool for routine public health surveillance. Below, we brief summarize most relevant packages and provide how ViroReportR is distinctively situated and filling a specific gap.

1. `EpiNow` and `EpiNow2`: both packages estimate time-varying reproduction number, growth rate and doubling time using a range of open-source tools. `EpiNow` is now deprecated in favor of `EpiNow2`. The aim of these packages is to provide a framework that is informed by community feedback and actively supported, avoiding limitations of naive implementation by the users. Forecasting is also available, by extending the approach used for estimation. However, they do not deliver a standardized multi-pathogen report intended for operational public health use.
2. `surveillance`: package focus on statistical methods for monitoring and modelling of time series of counts, proportion and categorical data, common in epidemiological settings. With a specific focus on aberration detection in count time series of communicable diseases, outbreak detection, and endemic-epidemic modelling framework, including visualization and simulation tools, this package doesn't offer short-term forecasts reporting tools.
3. `site_re`: it provides report templates for epidemiological surveys and outbreaks reports. The templates are intended for *Medecins Sans Frontieres* analyses, however, they are very generic and can be used by general epidemiological community. Their reports focuses on epidemiological indicators and standardize common epidemiological analyses, such as attack rates, descriptive analyses of outbreak data, it does  not provide short-term forecasts of daily counts across pathogens. 
4. `epi_predict`: it offers a framework for building transformation and forecasting pipelines for time series data tailored for epidemiological time-series data. It uses a varied of forecasting techniques and also provides backtesting for validation. However, it does not offer a tool for automated report generation, requiring the user to assembly their own reporting pipeline on top of `epi_predict`.
5. `PROF`: package designed for respiratory disease forecasting. `PROF` supports modelling of influenza and covid-19 using both compartmental modelling or a baseline statistical model. However, it doesn't generalized for arbitrary user-defined pathogens and does not include a unified automated reporting structure for surveillance purpose.

Across `CRAN` there are a variety of packages providing strong modelling capabilities, reporting templates, forecasting and outbreak detection. However, none of them integrates multiple component of the epidemiological analysis stack into a single automated workflow. 
`ViroReporR`:
1. accept multi-pathogen surveillance data in a format of daily counts(`date`, `confirm`, `disease_type`)
2. produces \(R_t)\-based forecasts with uncertainty quantification
3. performs backtesting for validation and summarizes standard metrics
4. renders a standardized HTML report for use on weekly briefs.
   
`ViroReportR` stands between low-level modelling libraries, such as `EpiNow2`, and `EpiEstim`, and reporting templates libraries (e.g. `site-re`), by integrating modelling, forecasting and reporting into one single tool. Filling a unique niche, `ViroReportR` offers an operationally tailored multi-pathogen reporting tool specifically for virus respiratory pathogens. 

# Software design

We want to provide a flexible and easy to use report generation tool, maintaining the overall structure of the report fixed. ViroReportR package works around a main function `generate_forecast_report()` that calls all other functions for data processing, fitting, forecasting, validation and render a final markdown file (`inst/vriforecasting_report.Rmd`).
The data input is fairly simple and consists of the daily counts of a respiratory viruses. Given that jurisdictions might be interested in different respiratory viruses, we allow users to add or remove viruses. All of this can be done by the data input. The required format of the data is 3 columns with `date`, `disease_type` and `confirm` (confirmed cases). If the user has added any additional viruses other than Influenza A, SARS-CoV2 and RSV, they will also need to add serial interval distribution parameters for those viruses. The overall text and explanations on the report are not dependent on results. Adding new virus provides more tabs, plots and summary tables on the final html report.  In addition, some respiratory viruses follow a clear respiratory season, whereas others may not. Thus, we include an option for users to provide seasonal period for each virus if needed. Currently, we provide estimations using the `EpiEstim` packages [@coriNewFrameworkSoftware2013], but we design `ViroReportR` to allow for inclusion of other model techniques in the future.
